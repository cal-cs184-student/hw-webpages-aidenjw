Problem Description

Realistic destruction effects are widely used in modern games, films, and simulation systems, but generating believable fractures remains challenging because it requires combining geometry processing, physically plausible fracture generation, rigid body dynamics, and real-time rendering. Traditional graphics pipelines usually treat meshes as static objects, so additional algorithms are needed to dynamically modify geometry when an object breaks. In this project, we aim to build an interactive destruction system that allows users to load a mesh, apply an impact, and observe realistic fracture behavior in real time. Our approach combines procedural mesh fracturing, using Voronoi-based or similar geometric partitioning methods, with rigid body simulation and OpenGL-based rendering to demonstrate how geometry algorithms and physics can be integrated into a responsive graphics application.

Goals and Deliverables

What You Plan to Deliver (Baseline)

Our baseline goal is to build a working interactive destruction demo in which a user can load a small set of clean triangle meshes, trigger a single fracture event, and observe the resulting fragments fall, collide, and settle in real time. To keep the project feasible within the course timeline, we plan to fracture each object once rather than support repeated re-fracturing, restrict ourselves to a controlled collection of input models, and use a lightweight physics engine such as Bullet for rigid body simulation instead of implementing a full solver from scratch. The final baseline system will include mesh loading, OpenGL rendering, camera controls, Voronoi-based or similarly procedural one-time fracture generation, fragment simulation under gravity with ground and inter-fragment collisions, and a polished demo scene that clearly shows the destruction effect from impact to rest.
