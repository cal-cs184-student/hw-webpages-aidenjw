Project Title

Summary: An interactive system for simulating the destruction of 3D objects uses procedural mesh fracturing and rigid body dynamics. When an object experiences an impact, it fractures into multiple dynamically-generated pieces that physically interact with the environment and each other. Our system will combine geometric algorithms for Constructive Solid Geometry (CSG) and Voronoi fracturing with rigid body simulation and real-time OpenGL rendering to produce visually realistic destruction effects.

Problem Description

Realistic destruction effects are widely used in modern games, films, and simulation systems. However, generating believable object fracture remains computationally challenging. Traditional graphics pipelines typically treat meshes as static objects stored in immutable vertex buffers. Simulating destruction requires dynamically modifying geometry, creating new topological structures, and seamlessly passing this new data to a physics solver in real time.

One common technique for destruction simulation is procedural fracturing, where a mesh is divided into fragments according to geometric patterns such as 3D Voronoi cells. Each fragment then becomes an independent rigid body subject to physical forces and collisions. While this technique produces visually convincing results, implementing it requires solving several technical challenges: robust mesh clipping algorithms, proper generation of new interior faces and normals, calculating accurate mass/inertia properties for procedurally-generated shapes, and interactive visualization using dynamic buffer updates.

In this project we aim to build an interactive C++ based destruction system that allows users to load an OBJ mesh, apply impacts, and observe realistic fracture behavior in real time. The system will demonstrate how geometric algorithms and physics simulation can be integrated into an interactive graphics application.

Goals and Deliverables

What You Plan to Deliver (Baseline)

Our primary goal is to implement a working interactive destruction simulator with the following architectural components:

Procedural Mesh Fracturing

We will implement Voronoi-based fracture generation for 3D triangle meshes.

Steps Include:
  • Seed Generation: Sampling seed points inside the mesh volume (using uniform random or Poisson-disk sampling).
  • Cell Computation: Computing 3D Voronoi cells. We will likely represent the boundaries between seeds as 3D splitting planes.
  • Mesh Clipping: Slicing the original mesh using the splitting planes via a mesh clipping algorithm (e.g., Sutherland-Hodgman for 3D polygons).
  • Topology Reconstruction: Generating new interior faces where the mesh was cut, ensuring the resulting fragments are closed, watertight manifolds. We will also compute new vertex normals for these interior faces to ensure proper lighting.

Rigid Body Simulation

Each watertight fragment will be converted into an independent rigid body. This requires dynamically computing the volume, center of mass, and inertia tensor of each newly generated mesh. The simulation will include:
  • Gravity and custom linear/angular velocity integration (e.g., Semi-implicit Euler integration).
  • Collision with a static ground plane and dynamic collision between fragments.
  • Physics Implementation Choice: We will either implement a custom rigid body solver (using algorithms like the Separating Axis Theorem (SAT) or GJK for convex-hull collision detection and impulse-based resolution) or integrate a robust C++ physics library such as Bullet Physics, passing our procedural fragments to the engine as btConvexHullShape objects.

Interactive Rendering

We will build an OpenGL-based renderer that supports:
  • Loading external .obj meshes and parsing them into a half-edge data structure for easy geometric manipulation.
  • Displaying fractured fragments by dynamically updating Vertex Buffer Objects or using instanced rendering for high fragment counts.
  • Real-time interactive camera controls (utilizing libraries like GLM and GLFW).
  • Time-stepping: The demo will allow users to move back and forth through timesteps to visualize the exact moments of fracture and splitting.

Evaluation

We will evaluate our system in several ways:
  • Performance Profiling: Measuring simulation frame rate (FPS) and memory allocation times versus the number of generated fragments to identify computational bottlenecks in the clipping phase
  • Visual Demonstration: Showcasing varying destruction scenarios, such as falling objects shattering upon hitting the ground, and impact-based destruction of a static mesh triggered by a user’s click.
  • Robustness Checks: Ensure generated fragments do not contain degenerate triangles or non-manifold edges.
Example scenes include:
  • A falling object breaking a mesh model several pieces as defined by our Voronoi splitting algorithm
  • Impact-based destruction of a loaded mesh (such as an object dropping to the ground by gravity and shattering, or a bullet going through a piece of glass)
  • Fragment collision and settling behavior

What You Hope to Deliver (Aspirational)

If time permits, we plan to extend the system with additional features:

Impact-Based Fracture

Instead of generating uniform fracture patterns, we will bias Voronoi seed placement toward the impact point so that more fragments appear near the collision location.

Progressive Fracture

Fragments could fracture again upon subsequent impacts, allowing multi-stage destruction.

Material Variation

We may experiment with different fracture behaviors for materials such as glass, ceramic, or stone by adjusting fragment distributions and break patterns.

Visual Effects
